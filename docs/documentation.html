What it is

A small, self-hosted utility that validates and corrects contact data (emails, phones, names, addresses, companies) in your own CSV/Excel files by calling the Foxentry REST API. It runs on your machine and writes results back to local files. ~1,900 lines of readable Python.

Architecture & data flow

The file never leaves the machine as a whole. The tool reads it row by row and sends only the field values needed for each validation to api.foxentry.com over TLS; results are written to local files in output/.

Technology & dependencies

  • Runtime: Python ≥ 3.9, cross-platform.
  • Network/TLS: Python standard library (urllib, ssl) — no third-party HTTP client; certificate verification on.
  • Formats: CSV via stdlib (zero deps); .xlsx via optional openpyxl (MIT). Without it, the tool degrades gracefully to CSV only.
  • UI: local web wizard on 127.0.0.1 (loopback only) or a --cli mode that opens no port.
  • Supply chain: one optional MIT dependency; no transitive third-party packages.

Privacy (GDPR)

Input/output files stay on your machine; the tool persists nothing beyond what you produce in output/. Only populated field values are sent (empty cells never leave). You remain the data controller; Foxentry acts as processor for validation queries under your existing agreement. No analytics or profiling.

Logs. By default the tool writes no request logs to logs/. You can enable logging for a single run on the final Order step; when enabled, the full request and response are saved (the API key in the headers stays masked) so you can inspect exactly what was sent and returned. Request logs are auto-pruned after LOG_RETENTION_DAYS (default 7) and can be wiped any time from the log viewer.

How to audit

  • Read the source — plain, commented Python under foxentry/; no obfuscation, no binaries.
  • Inspect egress — behind a proxy or with tcpdump, confirm the only host is api.foxentry.com.
  • Verify integrity — each release binary ships with a .sha256 checksum and a Sigstore build-provenance attestation (gh attestation verify <file> -R Foxentry/data-cleaner).

Verify it yourself

Run the tool behind an outbound firewall that allows only api.foxentry.com (and, if you use the optional Excel install, temporarily pypi.org + files.pythonhosted.org). This confirms that no other data leaves the machine. The source is readable and auditable; release binaries carry a .sha256 checksum and a Sigstore build-provenance attestation.

Air-gapped deployments. The optional “Install Excel” button is only a fallback — openpyxl is already bundled in vendor/. Set ALLOW_PIP_INSTALL=off in config.env to disable that button entirely (it then returns disabled without calling pip), leaving runtime egress strictly api.foxentry.com.

Using it

  • Start: run START.bat (Windows) or python run.py — the wizard opens in your browser at 127.0.0.1.
  • Map columns to data types (email, phone, name, address, company); leave others on “ignore”.
  • Estimate shows cost & time before anything is sent; you confirm to run.
  • Results land in output/ (cleaned file + HTML report). Original columns are kept; validation columns are added.
  • Stop/resume: close the window or Ctrl+C anytime — re-running resumes where it stopped, without re-charging done rows.
  • Encoding: CSV is read/written UTF-8 (BOM for Excel). Troubleshooting 401: check the API key and that it isn’t IP-restricted; the app uses API version 2.1.

Output file & results

The cleaner never overwrites your input. It writes a new file into output/ in the same format as the input (.csv stays CSV, .xlsx stays XLSX), plus a standalone HTML report. All original columns are kept unchanged; validation columns are appended after them, and any enrichment columns are added at the very end.

Column layout

For every validated field group (e.g. one address = street + city + ZIP) these columns are added, in this order:

ColumnMeaning
<group>_resultHuman-readable status of the record (see the status table below).
<column>_updatedThe value after correction — one per mapped column. If Foxentry corrected or normalised the value it is written here; otherwise it equals the original. Your original column stays untouched, so you can compare side by side.
<group>_suggestionAn alternative the API offers when the value is invalid but a likely candidate exists (e.g. a corrected e-mail). Empty when there is nothing to suggest.
<group>_noteWhy the record stayed invalid — the error description and which fields it relates to, e.g. Value is not valid. (streetWithNumber, city). Empty when the record ended valid.

“group” is the validation key (email, phone, name, location, company; or location1, location2… when you map several of the same type).

These are billed at the enriched rate (shown in the order step). Availability differs by country and source — fields the API doesn’t return for a given record simply stay empty.

Result statuses

Each _result cell holds one of the statuses below. “Fixed” means a corrected value is in _updated; “still invalid” means check _note (and _suggestion).

StatusMeaning & what to do
validCorrect as submitted. Nothing to do.
valid (corrected)Valid, but normalised (formatting, diacritics). Use _updated.
correctedWas invalid, fully fixed by Foxentry → use the value in _updated. This is data you’d otherwise lose to typos.
partially correctedPart was fixed, but the record as a whole is still invalid. _note says which field and why.
invalidInvalid and not fixable. See _note; consider dropping or manual review.
invalid (suggestion available)Invalid, but a likely candidate is in _suggestion — review before applying.
uncertain / uncertain (…)Foxentry could not determine validity (rare). Treat with caution; _suggestion/_updated may still help.
unverified / (empty)Not validated — the field was empty, so it was skipped (and not charged).
errorCommunication error with the API (e.g. a timeout). The row is counted as an error; re-run to retry just those.

Recommendations

  • Run a test batch first. Use the small test scope (or a custom row count) to sanity-check your mapping and pricing before the full run.
  • Map a country column when you have one (e.g. země → Country). Otherwise no country is sent; you can force a default in the address/company settings, but per-row country is more accurate for multi-country data.
  • Keep originals for comparison. Originals are never overwritten — diff <column> against <column>_updated to see exactly what changed.
  • Act by status: apply _updated for corrected; review _suggestion for suggestion available; read _note for partially corrected / invalid to decide drop vs. fix.
  • Use full data scope only when you need the extra columns (GPS, region, vocative, name days, legal form…) — it costs more per row.
  • Large files: raise concurrency in ⚙ Advanced for faster throughput; you can stop and resume anytime without re-charging finished rows.
  • Marketing payoff: not e-mailing invalid addresses protects your sender reputation; corrected postal addresses cut returned shipments. The report’s “Foxentry value” section quantifies the input vs. output error rate and how many records were rescued.

Co to je

Malý lokální nástroj, který validuje a opravuje kontaktní data (e‑maily, telefony, jména, adresy, firmy) ve vašich CSV/Excel souborech voláním REST API Foxentry. Běží u vás a výsledky zapisuje do lokálních souborů. ~1 900 řádků čitelného Pythonu.

Architektura a tok dat

Soubor jako celek nikdy neopustí stroj. Nástroj ho čte řádek po řádku a posílá jen hodnoty polí potřebné pro danou validaci na api.foxentry.com přes TLS; výsledky zapisuje lokálně do output/.

Technologie a závislosti

  • Běhové prostředí: Python ≥ 3.9, multiplatformní.
  • Síť/TLS: standardní knihovna Pythonu (urllib, ssl) — žádný cizí HTTP klient; ověřování certifikátu zapnuté.
  • Formáty: CSV přes stdlib (bez závislostí); .xlsx přes volitelný openpyxl (MIT). Bez něj nástroj funguje jen s CSV.
  • UI: lokální webový průvodce na 127.0.0.1 (jen loopback) nebo režim --cli bez otevřeného portu.
  • Dodavatelský řetězec: jedna volitelná MIT závislost; žádné tranzitivní balíčky.

Doporučení

  • Nejdřív testovací várka. Použijte malý testovací rozsah (nebo vlastní počet řádků) a zkontrolujte mapování i cenu před plným během.
  • Namapujte sloupec se zemí, pokud ho máte (např. země → Country). Jinak se žádná země neposílá; v nastavení adresy/firmy lze vynutit výchozí zemi, ale u vícezemních dat je přesnější země per řádek.
  • Originály pro porovnání. Originály se nikdy nepřepisují — porovnejte <sloupec> vs. <sloupec>_updated a vidíte přesně, co se změnilo.
  • Postupujte podle stavu: u opraveno použijte _updated; u existuje návrh zkontrolujte _suggestion; u částečně opraveno / neplatné přečtěte _note a rozhodněte vyřadit vs. opravit.
  • Full datascope používejte jen když potřebujete sloupce navíc (GPS, kraj, 5. pád, jmeniny, právní forma…) — stojí víc za řádek.
  • Velké soubory: v ⚙ Advanced zvyšte souběžnost pro vyšší propustnost; běh lze kdykoli zastavit a navázat bez opakovaného účtování hotových řádků.
  • Marketingový přínos: neposílání na nevalidní e‑maily chrání reputaci odesílatele, opravené poštovní adresy snižují vratkovost. Sekce „Přínos Foxentry" v reportu vyčíslí chybovost na vstupu vs. výstupu a kolik záznamů se zachránilo.
